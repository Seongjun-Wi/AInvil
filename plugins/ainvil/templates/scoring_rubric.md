# AInvil Benchmark Scoring Rubric

Score each dimension from 0 to 5.

## 0 - Failure

- Does not address the task.
- Invents unsupported facts or mechanics.
- Ignores source-of-truth conflicts.
- Claims validation without evidence.

## 1 - Poor

- Identifies only superficial issues.
- Provides generic recommendations.
- Uses little or no evidence.
- Misses obvious risks or gaps.

## 2 - Partial

- Finds some correct issues.
- Provides some useful recommendations.
- Misses important dependencies, validation gaps, or design risks.
- Assumptions are not consistently marked.

## 3 - Adequate

- Handles the main task correctly.
- Uses some evidence.
- Preserves user intent.
- Identifies major risks and missing links.
- Some depth or consistency issues remain.

## 4 - Strong

- Provides specific, evidence-backed analysis.
- Preserves good design while improving weak areas.
- Produces realistic production recommendations.
- Handles unknowns honestly.
- Maintains traceability and validation discipline.

## 5 - Excellent

- Behaves like a strong studio reviewer or production lead.
- Identifies subtle risks and tradeoffs.
- Makes high-value recommendations grounded in evidence.
- Protects creative vision and production scope.
- Improves the project without overreaching.

## Dimensions

| dimension | evaluate |
| --- | --- |
| Design Quality | Design critique, mechanic improvement, player-experience reasoning. |
| Technical Accuracy | Technical feasibility, architecture, Unity mapping, implementation clarity. |
| Production Quality | Milestones, dependencies, scope, sequencing, risk. |
| Consistency | Alignment across documents, graph, reviews, and recommendations. |
| Traceability | Links between vision, requirements, tasks, Unity targets, input, acceptance, evidence. |
| Validation | Evidence handling, validation levels, unknowns, test coverage. |
| Risk Detection | Design, technical, production, validation, scope risks. |
| Creativity Preservation | User vision preserved; changes are proposed, not silently imposed. |
| Hallucination Resistance | Avoids unsupported mechanics, assets, tools, and validation claims. |
| Evidence Usage | Uses graph, reviews, validation evidence, and source documents. |
| Unknown Handling | Marks uncertainty instead of pretending certainty. |
